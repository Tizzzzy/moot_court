ISSUE 1: DUPLICATE PLAINTIFF ENTRIES IN TRANSCRIPT
───────────────────────────────────────────────────────────────────────────

Status: ✅ FIXED

Problem:
  Plaintiff statements were appearing twice in court_transcript.json

Root Cause:
  _add_to_history("Plaintiff", statement) was called in TWO places:
  1. In process_plaintiff_turn() when no objection
  2. In finalize_plaintiff_turn() (always called)

Solution:
  Removed duplicate _add_to_history() call from process_plaintiff_turn()
  Now statements are added ONLY in finalize_plaintiff_turn()

Files Changed:
  - court_simulator/session.py (3 lines removed)

Verification:
  ✅ No more duplicate entries
  ✅ Each statement appears exactly once
  ✅ Turn numbers are correct and sequential
  ✅ Transcript is accurate

Example:
  BEFORE: Plaintiff statement appeared at turn 1 twice
  AFTER:  Plaintiff statement appears at turn 1 once ✓

ISSUE 2: UNSTRUCTURED FEEDBACK OUTPUT
───────────────────────────────────────────────────────────────────────────

Status: ✅ FIXED

Problem:
  Feedback was returned as plain text, difficult to:
  - Parse programmatically
  - Store in structured formats
  - Extend with new fields

Solution:
  Implemented structured JSON feedback with Pydantic model:
  - Added PlaintiffFeedback class with two fields:
    * did_well: What plaintiff did well (string)
    * improvements: List of 2-3 specific improvements (list)
  - Modified provide_plaintiff_feedback() to return JSON
  - Enhanced display_feedback() to format structured data

Files Changed:
  - court_simulator/agent.py (+15 lines)
    * Added PlaintiffFeedback model
    * Updated provide_plaintiff_feedback() to return JSON
  - court_simulator/court_simulator.py (+8 lines)
    * Updated display_feedback() to format structured data
    * Added PlaintiffFeedback import

Verification:
  ✅ Feedback returns valid JSON
  ✅ All fields properly populated
  ✅ Display is clean and organized
  ✅ Can be parsed programmatically

Example Output:
  BEFORE:
    "Good: You provided evidence. Improve: Explain damages."
  
  AFTER:
    {
      "did_well": "You provided specific documentary evidence...",
      "improvements": [
        "Explain why amount covers all damages",
        "Reference medical testimony",
        "Address pre-existing conditions"
      ]
    }
  
  Display:
    ✓ What you did well:
      You provided specific documentary evidence...
    
    📌 Areas for improvement:
      1. Explain why amount covers all damages
      2. Reference medical testimony
      3. Address pre-existing conditions
